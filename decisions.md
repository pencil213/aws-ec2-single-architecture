# 設計判断ログ

## 1. なぜEC2を停止したか
学習用途のため、常時稼働は不要と判断した。
AWSの課金は稼働時間に依存するため、使用しない時間は停止することで
無駄なコストを防ぐことができる。

## 2. なぜUTMでもLinuxを学習するか
AWSはネットワーク設定やセキュリティの理解に集中したいため、
Linuxの基本操作はローカル環境（UTM）で事前に習得する方が効率的と判断した。

## 3. なぜ最初から高可用構成にしなかったか
設計を理解せずに冗長構成を組んでも意味がないため、
まずは単一構成でEC2の挙動を正確に理解することを優先した。

## 4. 今後の設計方針
- EC2単体の挙動を理解した後にALB + 複数EC2構成へ発展させる
- IAMロールを付与し、鍵管理を減らす
