# AWS EC2 Single Architecture

## 目的
AWSとLinuxの基礎学習を目的として、最小構成のEC2環境を設計・構築した。

## 要件
- 利用者：自分のみ
- 想定同時接続数：1
- 可用性：学習用途のため停止しても問題なし
- セキュリティ：SSH接続を安全に行えること
- コスト：できるだけ無料枠内に収める
- 運用：1人で管理、常時監視しない

## 採用構成
- VPC（デフォルト）
- EC2（Amazon Linux 2023 / t系インスタンス）
- セキュリティグループ
  - SSH(22)を自分のIPアドレスからのみ許可
- キーペアによる認証

## 設計上の判断理由
### なぜEC2を使ったか
Linux操作とサーバー管理の基礎を理解するため、仮想マシンとして最も基本的なEC2を採用した。

### なぜ単一構成にしたか
可用性要件が低く、障害時に手動復旧できるため
ALBやマルチAZは今回は不要と判断した。

### なぜSSHを自分のIPのみに制限したか
不特定多数からのアクセスを防ぐため、セキュリティグループで
自分のグローバルIPのみ許可した。

## 想定されるトラブル
- EC2が停止した場合：手動で起動する
- SSH鍵を紛失した場合：新しいキーペアでインスタンスを再作成
- 課金が発生した場合：インスタンス停止または削除
